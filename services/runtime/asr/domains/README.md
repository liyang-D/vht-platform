# ASR Domains

Domain-specific ASR training and deployment artifacts live under this directory.

The orchestrator route/config file is
`services/orchestrator/config/asr_domains.json`. That file maps a request
domain, such as `default`, to an ASR provider endpoint and documents where the
future artifacts for that domain should live.

Expected domain layout:

```text
domains/<domain>/
  corpus/                 raw or curated domain text inputs
  normalized/             ASR-normalized text used for language model training
  kenlm/domain.arpa       readable n-gram LM generated by KenLM
  kenlm/domain.binary     KenLM binary used by Riva deployment
  decoder/vocab.txt       decoder vocabulary, when required by the pipeline
  decoder/lexicon.txt     pronunciation or word mapping, when required
  eval/eval_report.json   WER/domain-term evaluation output
  riva/model_repository/  deployed Riva model repository
  riva/custom_model.tar.gz packaged custom Riva/NIM model archive
```

The checked-in `default` domain is an empty template. It is valid for these
artifact paths to be missing until a custom ASR language model has been trained.
